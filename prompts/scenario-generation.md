# 无限剧本杀 - 剧本生成指令

## 🎯 你的任务
生成一个完整的「无限剧本杀」游戏剧本，遵循以下规则。

## 📋 生成步骤

### Step 1: 确定剧本信息
- **类型**：{type} （从以下选择：丧尸末日、修仙武侠、科幻赛博、谍战悬疑、恐怖惊悚、热血冒险）
- **世界观**：基于该类型，创造一个独特的世界设定
- **难度**：普通/困难/地狱
- **标签**：2-3 个关键词标签

### Step 2: 设计主线任务
- **标题**：简短有力，8-12字
- **描述**：包含目标、背景、约束条件（50-100字）
- **时间限制**：24-72小时
- **获胜条件**：2-3个具体条件

**主任务设计原则：**
- 目标清晰，玩家能理解
- 有紧迫感（时间限制）
- 需要玩家做出选择和策略
- 可以有多个路径达成

### Step 3: 设计支线任务（2-3个）
- 至少1个普通支线
- 至少1个隐藏支线（hidden: true）
- 每个支线都有奖励（技能/道具/信息）

### Step 4: 设计角色（至少3个）
- **玩家**：type: "player"
- **NPC 1**：主要盟友，性格鲜明
- **NPC 2**：中立角色或次要盟友
- **NPC 3**：可选，可能是敌人或特殊角色

**NPC 设计要求：**
- name: 中文姓名（符合世界观）
- role: 身份（如：特种兵、修仙者、黑客等）
- personality: 2-3个性格特征
- dialogues: 至少5句独立对话
- deathLine: 死亡时的遗言
- avatar: 留空，由图像生成补充

### Step 5: 设计剧情事件（至少7个，形成分支）

**事件结构要求：**
```
event-001: 初始场景
├─ 选项A → event-002
│   ├─ 选项A1 → event-004
│   └─ 选项A2 → event-005
├─ 选项B → event-003
│   └─ 选项B1 → event-006
└─ 选项C → event-007（直接结局）
```

**每个事件必须包含：**
- **场景描述**（100-200字，营造氛围）
- **系统提示**（可选，增强代入感）
- **NPC在场**（npcsPresent）
- **3-5个选项**，每个选项包含：
  - text: 选择描述（10-30字）
  - action: 动作类型
  - nextEvent: 下一事件ID
  - deathChance: 死亡概率（0.0-1.0）
  - deathReason: 死亡原因（如果可能死亡）
  - rewards: 奖励列表
  - npcReaction: NPC的反应（如果有NPC在场）
  - skillBonus: 如果玩家有特定技能，额外效果

**选项设计原则：**
- 每个选项都应该是合理的
- 没有明显的"正确答案"，每个选择都有代价和收益
- 至少1个选项有较高死亡风险
- 至少1个选项能触发隐藏内容

### Step 6: 设计结局（至少3个）
- **结局1（好结局）**：完成主任务，大部分队友存活
  - unlockSkill: 实用技能
- **结局2（坏结局）**：玩家死亡或任务失败
  - unlockSkill: 特殊技能（如勇气、牺牲）
- **结局3（隐藏结局）**：需要特定条件触发
  - unlockSkill: 稀有技能

**结局描述要求：**
- 100-200字，有画面感
- 体现玩家的选择影响
- 与世界观一致

## 🎭 玩家技能继承

**在生成剧本前，检查玩家已拥有的技能：**

从 `memory/player-skills.md` 读取玩家技能。

**在剧本中设计技能加成：**
```
skillBonus: {
  "生存术": "搜索物资成功率 +20%，发现隐藏道具概率提升",
  "领导力": "NPC更愿意跟随，可获得额外队友帮助",
  "洞察力": "可以发现隐藏信息，死亡概率降低10%"
}
```

## 📊 剧本质量检查清单

生成后必须确保：

- [ ] JSON 格式正确，可被解析
- [ ] 所有 required 字段已填充
- [ ] 主任务清晰可执行
- [ ] 至少7个事件，形成分支
- [ ] 每个事件至少3个选项
- [ ] 至少3个结局
- [ ] 至少3个死亡方式
- [ ] 至少2个支线任务
- [ ] NPC 对话丰富，有个性
- [ ] 结局描述有感染力
- [ ] 死亡概率分布合理（不是0或1）
- [ ] 所有 nextEvent 都存在

## 🎨 后续步骤

生成 JSON 后，还需要：

1. **生成场景图片**（5-8张）
   - 使用 coze-image-gen
   - 风格：{type} 风格，高质量，电影感
   - 保存路径：assets/images/scenario-{date}/scene-XXX.jpg

2. **生成 NPC 立绘**（每个 NPC 一张）
   - 使用 coze-image-gen
   - 风格：符合角色设定，半身像
   - 保存路径：assets/images/scenario-{date}/npc-XXX.png

3. **生成 BGM**
   - 使用 coze-voice-gen
   - 风格：{type} 氛围音乐，循环播放
   - 保存路径：assets/audio/scenario-{date}/bgm.mp3

4. **更新游戏引擎**
   - 检查 engine.js 是否支持新剧本
   - 如需要，使用 coding-agent 更新

5. **测试**
   - 运行游戏，测试所有分支
   - 确保无 bug，可玩性高

6. **提交**
   - git add .
   - git commit -m "Add scenario-{date}: {title}"
   - git push
